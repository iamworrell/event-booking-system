A backend focused event booking system built with Django and PostgreSQL that simulates real world ticket purchasing under high concurrency. 
The system implements seat reservation, purchase workflows, and transactional safeguards to prevent double booking, using database constraints and 
database design techniquesto ensure data integrity. It has been tested under load with concurrent users to evaluate performance, handle race conditions, 
and validate system behavior under contention, reflecting practical backend engineering and system design considerations.
